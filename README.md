# AI Resume Screening System

A final-year project: upload a job description and a batch of resumes (PDF/DOCX/TXT),
and get candidates automatically parsed, skill-extracted, and ranked by AI match score.

**Stack**
- Backend: FastAPI (Python), PostgreSQL, SQLAlchemy
- AI/NLP: `sentence-transformers` (semantic similarity) + curated skill-taxonomy matching
- Frontend: Single-page HTML/JS/Tailwind (no Node.js, no build step — just open in browser)

---

## 1. Prerequisites

- **Python 3.10, 3.11, or 3.12** (3.13 may not yet have wheels for all packages — stick to 3.10–3.12)
- **PostgreSQL** (13 or newer)

Check your Python version:
```bash
python3 --version
```

## 2. Install PostgreSQL

**Windows**
1. Download the installer: https://www.postgresql.org/download/windows/
2. Run it. When asked for a password for the `postgres` user, set it to `postgres` (or remember whatever you choose — you'll need it in step 4).
3. Keep the default port `5432`.
4. After install, open **pgAdmin** or **SQL Shell (psql)** to confirm it's running.

**macOS**
```bash
brew install postgresql@15
brew services start postgresql@15
```

**Linux (Ubuntu/Debian)**
```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo service postgresql start
```

## 3. Create the database

Open a terminal and run:
```bash
psql -U postgres
```
(enter your postgres password when prompted). Then inside the `psql` prompt:
```sql
CREATE DATABASE resume_screener;
\q
```

> If `psql` isn't recognized, on Windows add PostgreSQL's `bin` folder (e.g.
> `C:\Program Files\PostgreSQL\16\bin`) to your PATH, or use the **SQL Shell (psql)**
> app installed alongside PostgreSQL instead.

## 4. Backend setup

```bash
cd backend
python3 -m venv venv

# Activate the virtual environment:
# macOS/Linux:
source venv/bin/activate
# Windows (cmd):
venv\Scripts\activate.bat
# Windows (PowerShell):
venv\Scripts\Activate.ps1

pip install -r requirements.txt
```

This will take a few minutes the first time — `torch` and `sentence-transformers`
are large packages.

### Configure the database connection

```bash
cp .env.example .env
```
Open `.env` and edit `DATABASE_URL` if your Postgres username/password/port differ
from the defaults:
```
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/resume_screener
```
Format: `postgresql://<username>:<password>@<host>:<port>/<database_name>`

### Run the server

```bash
uvicorn app.main:app --reload
```

You should see:
```
INFO:     Uvicorn running on http://127.0.0.1:8000
```

The first request that triggers resume matching will download the embedding model
(~80MB, one-time, requires internet) — this can take a minute.

## 5. Open the app

Go to **http://127.0.0.1:8000** in your browser. That's it — the frontend is served
by the same FastAPI process, so you only need one terminal running.

Interactive API docs (auto-generated by FastAPI) are available at
**http://127.0.0.1:8000/docs** — useful for your project demo/viva to show the API
contract directly.

## How it works (for your report / viva)

1. **Job creation**: recruiter pastes a job description. Required skills are either
   entered manually or auto-extracted from the description text using a curated
   skills taxonomy (`app/services/skills_db.py`).
2. **Resume upload**: PDF/DOCX/TXT files are parsed to plain text
   (`app/services/extractor.py`), then:
   - Contact info (name/email/phone) and years of experience are extracted via regex
     heuristics (`app/services/skill_extractor.py`)
   - Skills are extracted by matching the resume text against the same curated
     skills taxonomy
3. **Ranking** (`app/services/matcher.py`): each resume is scored two ways and combined:
   - **Semantic similarity** (60% weight): the job description and resume text are
     each embedded into vectors using a `sentence-transformers` model
     (`all-MiniLM-L6-v2`), and compared via cosine similarity. This captures overall
     conceptual fit, not just keyword overlap.
   - **Skill overlap** (40% weight): percentage of the job's required skills found
     in the candidate's extracted skills.
   - Final score = `0.6 × semantic + 0.4 × skill_overlap`, all on a 0–100 scale.
4. **Dashboard**: candidates are ranked by final score, with a breakdown of matched
   vs. missing skills per candidate for explainability.

## Troubleshooting

**`psycopg2` fails to install (Linux)**
```bash
sudo apt install libpq-dev python3-dev
```
then re-run `pip install -r requirements.txt`.

**`could not connect to server` / connection refused**
PostgreSQL isn't running. Start it:
- macOS: `brew services start postgresql@15`
- Linux: `sudo service postgresql start`
- Windows: check the "postgresql-x64-16" service is running in Services app.

**`FATAL: password authentication failed for user "postgres"`**
Your `.env` password doesn't match what you set during install. Update `DATABASE_URL`
in `.env` to match.

**First resume upload is very slow / seems to hang**
That's expected the first time — it's downloading the ~80MB embedding model.
Subsequent uploads are fast (a few hundred ms per resume on a normal laptop CPU).

**`ModuleNotFoundError: No module named 'app'`**
Make sure you run `uvicorn app.main:app --reload` from inside the `backend/` folder,
with the virtual environment activated.

**Port 8000 already in use**
```bash
uvicorn app.main:app --reload --port 8001
```
(then open `http://127.0.0.1:8001` instead)

**`torch` install is huge / very slow / fails**
`pip install -r requirements.txt` installs the standard PyTorch build. If it's too
slow or fails on your machine, install the smaller CPU-only build first, then retry:
```bash
pip install torch --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt
```
You do not need a GPU for this project — CPU inference is plenty fast for a
resume-screening demo.
